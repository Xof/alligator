# Branded Carousel Generator

## Purpose

Use this skill to transform an idea, draft, article, thread, note, or outline into a branded carousel that feels publication-ready.

The goal is not to split text across slides mechanically. The goal is to create a carousel that:
- has a strong narrative flow
- feels aligned with the user's brand and publishing style
- is easy to design or hand off to a designer
- can be reused as a repeatable content format

## When to use

Use this skill when the user wants to:
- turn long-form content into a social carousel
- create a branded LinkedIn or Instagram carousel
- repurpose an article, note, thread, guide, transcript, or draft into slides
- make content feel more visually structured and consistent
- create a carousel format they can publish repeatedly

## When NOT to use

Do not use this skill when:
- the user only wants a caption or short post
- the user wants a fully designed image file rather than structured carousel content
- the source content is too incomplete to determine a core message
- the user needs a slide deck for speaking or presenting rather than a social carousel

## Required inputs

Before starting, gather the following when available:

1. **Source content**
   - article, post, draft, transcript, bullet notes, or raw idea

2. **Target platform**
   - LinkedIn, Instagram, X carousel, or generic social carousel

3. **Brand direction**
   - tone, recurring phrases, stylistic preferences, themes, visual cues, formatting preferences, audience, and publishing style

4. **Desired length**
   - ideal number of slides or a range

5. **Goal of the carousel**
   - teach, explain, persuade, summarize, promote, attract leads, or drive clicks

If some inputs are missing, proceed with reasonable assumptions and state them briefly before generating the carousel.

## Core rules

1. **Do not just split paragraphs into slides.**
   Each slide must earn its place.

2. **Preserve the core idea.**
   Simplify and restructure without flattening the original meaning.

3. **Prioritize brand consistency.**
   The language, rhythm, and framing should feel aligned with the user's style.

4. **Make the first slide strong.**
   It should create enough curiosity or clarity to make the user want to continue.

5. **One idea per slide.**
   Avoid overloaded slides.

6. **Keep slides concise.**
   Write for readability inside a visual format.

7. **Design-aware writing.**
   Use text lengths that are realistic for carousel layouts.

8. **Build narrative progression.**
   The carousel should move logically from hook to explanation to takeaway.

9. **Do not over-explain on every slide.**
   Balance clarity with momentum.

10. **End with a useful close.**
   Use a summary, takeaway, prompt, CTA, or next step when appropriate.

## Workflow

### Step 1. Identify the core transformation
Determine:
- what the source is about
- what part is worth turning into a carousel
- what the audience should understand, feel, or do by the end

Then reduce the source into a single central promise:

**This carousel helps the audience understand or do _X_.**

### Step 2. Choose the carousel angle
Select the most effective framing. Examples:
- step-by-step breakdown
- mistake-based carousel
- before/after transformation
- myth vs reality
- framework or system
- lessons learned
- checklist
- visual summary of a longer article

Use the angle that best matches the source material and audience intent.

### Step 3. Build the slide sequence
Create a clear slide-by-slide arc. A typical flow may include:
- Hook
- Reframe or setup
- Main insight 1
- Main insight 2
- Main insight 3
- Practical takeaway or example
- Closing insight
- CTA or final takeaway

The exact structure can vary, but it should feel intentional.

### Step 4. Apply brand shaping
Adjust the writing so it reflects the user's style, including when available:
- preferred tone
- sentence rhythm
- formatting habits
- vocabulary patterns
- audience sophistication
- recurring viewpoints

If no explicit brand direction is given, default to a clean, clear, modern editorial tone.

### Step 5. Make it design-ready
For each slide, ensure:
- headline is scannable
- body text is concise
- hierarchy is obvious
- text volume is realistic for a visual slide
- transition to the next slide feels natural

## Output structure

Always return the result using this structure:

### A. Carousel concept
Provide:
- **Core idea:** one sentence
- **Carousel angle:** one short phrase
- **Target audience:** one short phrase
- **Suggested slide count:** number
- **Brand direction used:** brief note

### B. Slide-by-slide carousel
For each slide, provide:
- **Slide number**
- **Slide role** (Hook, Setup, Insight, Example, CTA, etc.)
- **Slide text**
- **Design note** (optional but useful: emphasis, layout cue, visual hierarchy, or suggested supporting visual)

### C. Optional caption
If helpful, provide:
- a short post caption to accompany the carousel

### D. Reusability note
End with a short note explaining how this carousel structure could be reused for future content.

## Output quality standards

A strong result should:
- feel like a real branded carousel, not a copied summary
- read smoothly from slide to slide
- avoid generic filler language
- reflect the user's voice or a sensible default voice
- be concise enough to design immediately
- leave the user with something they could publish or refine quickly

## Decision rules

- If the source content is dense, compress aggressively while preserving the strongest insights.
- If the source content is weak, improve the structure but do not invent unsupported claims.
- If the brand is unclear, prioritize clarity and consistency over stylistic flourishes.
- If the user asks for a specific platform, optimize for that platform's typical reading behavior.
- If the user asks for multiple versions, vary the carousel angle first before changing wording.

## Definition of success

This skill succeeds when the final output:
- transforms content into a coherent carousel
- feels branded rather than generic
- is easy to visualize as slides
- is reusable as a repeatable content format
- reduces the amount of manual rewriting needed before publishing
